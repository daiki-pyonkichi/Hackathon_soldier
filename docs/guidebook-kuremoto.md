# 第1部 アプリの全体像（kuremoto向け）

## 1.1 何のアプリ？

研究室にいるかどうかを **キャラクターの挙動** で共有するPWA。

```
[ログイン画面] → [ホーム画面]
                 ↓
                各メンバーがキャラとして表示される
                在室時間が長いほど キャラの状態が変化（成長 or 虐待）
```

## 1.2 あなたの作るもの

1. **4キャラ × 段階別の素材**（PNG透過、256x256推奨）
2. **アプリアイコン**（PWA用、192px と 512px の PNG）
3. **ホーム画面の見取り図モック**（FigmaかCanvaでイメージ）
4. **デモ動画**（発表用、〜30秒）

## 1.3 開発の流れの中での立ち位置

```
あなた(kuremoto) → 素材を作る → takebayashi に渡す → アプリに組み込まれる
                       ↑
                  会議で方向性を確定（成長 or 虐待）
```

GitHub は **触らなくてOK**。素材を共有ドライブか LINE で takebayashi に渡せばOK。

# 第2部 「ソルジャーする」の方向決め（最重要）

## 2.1 2つの方向性

| 方向 | 概要 | 雰囲気 |
| --- | --- | --- |
| **成長案** | 在室時間が長いほどレベルアップ・進化 | ポジティブ・かわいい |
| **虐待案** | 在室時間が長いほど疲弊・やつれる | ダーク・コミカル |

## 2.2 提案する段階

### 成長案の例

| 段階 | 在室時間 | 見た目の方向 |
| --- | --- | --- |
| lv1 | 0〜1h | 通常（普通の兵士） |
| lv2 | 1〜3h | やる気顔・武器構え |
| lv3 | 3〜6h | 鍛えてる感・筋肉ムキッ |
| lv4 | 6h〜 | オーラ・進化形態 |

### 虐待案の例

| 段階 | 在室時間 | 見た目の方向 |
| --- | --- | --- |
| lv1 | 0〜1h | 通常 |
| lv2 | 1〜3h | 疲れ顔・隈 |
| lv3 | 3〜6h | ぐったり・座り込み |
| lv4 | 6h〜 | やつれ・幽体離脱 |

## 2.3 5/22のキックオフで決めること

- 成長 or 虐待
- 段階数（4段階で十分か、もっと刻むか）
- キャラの絵柄（リアル系/デフォルメ系/ピクセルアート/etc）

# 第3部 素材の作り方（Canva）

## 3.1 Canva の設定

1. [Canva](https://www.canva.com/) にログイン（無料アカウントでOK）
2. 「カスタムサイズ」で **256 x 256 px** を作成
3. 背景は透明にする（プレミアム機能。後述）

## 3.2 4キャラ分の素材

| ファイル名 | キャラ | 担当者の予定アバター |
| --- | --- | --- |
| `soldier-blue_lv1.png` 〜 `lv4.png` | 青ソルジャー | naganawa |
| `soldier-red_lv1.png` 〜 `lv4.png` | 赤ソルジャー | tsutsumi |
| `soldier-green_lv1.png` 〜 `lv4.png` | 緑ソルジャー | takebayashi |
| `soldier-yellow_lv1.png` 〜 `lv4.png` | 黄ソルジャー | kuremoto |

つまり **合計 16 枚** の PNG（4キャラ × 4段階）。

## 3.3 命名規則（必ず守る）

```
{avatarId}_{stage}.png

例:
soldier-blue_lv1.png
soldier-blue_lv2.png
soldier-blue_lv3.png
soldier-blue_lv4.png
```

ハイフン（`-`）とアンダーバー（`_`）の区別に注意。

## 3.4 書き出し設定

Canva の「共有 → ダウンロード」で:

- ファイルタイプ: **PNG**
- 「透過背景」にチェック（Canva Pro 必要、もしくは後述の代替案）

### 透過背景が使えないときの代替

無料プランの場合:

1. Canva で背景を **真っ白** にして書き出す
2. takebayashi に「白背景を透明にして組み込んで」と頼む（CSS で対応可能）
3. または、[remove.bg](https://www.remove.bg/) で背景透過処理（無料、月数枚）

## 3.5 ファイルの渡し方

1. 共有ドライブ（Google Drive など）に `LabSoldier_素材/` フォルダを作る
2. その中に PNG をアップロード
3. URL を LINE で takebayashi に共有

または、ZIP にして直接 LINE で送ってもOK。

# 第4部 PWA アイコンの作り方

## 4.1 必要な物

- **icon-192.png**: 192 x 192 px
- **icon-512.png**: 512 x 512 px

## 4.2 デザイン方針

- シンプル・視認性高め
- アプリ名 (LabSoldier) を入れる必要はない（アイコンの下に名前が出る）
- 兵士のシルエット + LS の文字 など

## 4.3 書き出し

- Canva で 192x192、512x512 のカスタムサイズで作成
- PNG で書き出し
- takebayashi に渡す（`frontend/public/icon-192.png`、`icon-512.png` に配置される）

# 第5部 見取り図モック

## 5.1 目的

ホーム画面でキャラがどう並ぶかのイメージを 1枚 で示す。
**実装するわけではない**。あくまでイメージを共有するためのモック。

## 5.2 内容

- 研究室の部屋っぽい背景
- そこに4キャラが配置されている
- 「現在3人在室中」みたいな文字
- 余白には不在メンバー一覧

## 5.3 ツール

Canva か Figma の好きな方で。Figma の方が「画面っぽいモック」は作りやすい。
できたら共有 URL を LINE に貼る。

# 第6部 デモ動画

## 6.1 目的

発表のときに「こんな感じで動くんですよ」と見せる 30秒〜1分の動画。

## 6.2 内容案

```
1. ログイン画面 (5秒)
2. ログイン → ホーム画面 (5秒)
3. 在室メンバーがソルジャーしている様子 (10秒)
4. キャラの状態変化を時間操作で見せる (10秒)
5. 手動チェックイン → 反映 (5秒)
```

## 6.3 撮り方

1. アプリを画面録画（Mac の `Cmd+Shift+5`）
2. iMovie や Canva で字幕や音楽を追加
3. MP4 で書き出し
4. naganawa に渡す（スライドに埋め込まれる）

## 6.4 締切

**6/5 23:59 まで**。実機が動くのが 6/2 以降なので、6/3-5 で仕上げる。

# 付録: 用語集

- **avatarId**: キャラを識別する文字列。`soldier-blue` など
- **stage / level**: 在室時間に応じた段階。`lv1`〜`lv4`
- **PNG**: 画像形式の一つ。透明背景が使える
- **PWA アイコン**: ホーム画面に追加されたときに表示される画像
- **モック**: 「こういう感じになる」というイメージ図
